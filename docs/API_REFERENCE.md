# API Reference

> **Verification checklist:** generated by grepping each file in
> `backend/src/routes/*.ts` for `.get(`/`.post(`/`.put(`/`.patch(`/`.delete(`
> and `requireAuth`/`requireStaffAuth` call sites, then reading
> `backend/src/routes/index.ts` for the CORS/mount wiring. Route params
> (`:botId` etc.) are taken from the literal path strings in each file. If a
> route was added after 2026-07-20, it won't be here — re-grep
> `backend/src/routes/`.

Auth legend: **public** = no auth middleware · **auth** = `requireAuth`
(customer Cognito pool, `c.get('user').sub` = `clientId`) · **staff** =
`requireStaffAuth` (separate staff Cognito pool).

Base mount points, from `backend/src/routes/index.ts`:
`/api/admin`, `/api/auth`, `/api/bots`, `/api/chat`, `/api/leads`,
`/api/kb`, `/api/clients`, `/api/forms`, `/api/integrations`,
`/api/voice-agents`, `/api/webhooks`.

## Health check

`GET /health` — outside all `/api/*` mounts, no CORS config applied, used as
the deploy workflow's post-deploy smoke test (`deploy.yml`, "Health check
main Lambda" step hits `/api/bots/health-check/config` specifically, not
`/health` — worth noting these are two different endpoints).

## Summary: how far the real public surface has grown

The in-repo `CLAUDE.md` states, as an "IMMUTABLE" rule: *"All routes except
`/api/chat` and `/api/bots/:id/config` require Cognito JWT auth."* The
actual public surface today is at least 13 routes across 6 route files
(chat's 3, bots' 1, leads' 1, forms' 2, voice-agents' 4, webhooks' 1, plus
`/api/auth/confirm` and the Zoho OAuth callback) — an expected consequence
of adding public-facing widgets (form, voice) and webhooks after that rule
was written, not a violation to "fix," but worth updating the rule's wording
next time `CLAUDE.md` is touched so it describes reality.
